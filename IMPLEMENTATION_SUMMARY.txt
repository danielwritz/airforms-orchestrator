HOW TO RUN THE ORCHESTRATOR LOCALLY

Prerequisites:
- Node.js installed (recommended LTS)

From workspace root:
1) Install orchestrator dependencies
   cd airforms-orchestrator
   npm install

2) Run tests
   npm test

3) Build
   npm run build

4) Start server
   npm start

Server default:
- http://localhost:3000
- Endpoint: POST /turn

QUICK MANUAL TEST (POWERSHELL)

1) Initial user_text turn:
Invoke-RestMethod -Method Post -Uri "http://localhost:3000/turn" -ContentType "application/json" -Body '{"conversationId":"c_demo","message":{"type":"user_text","text":"I want to check my insurance."}}'

Expected result:
- assistant_message asking for policy details
- ui_frame with frameId insurance:lookup and primaryAction ui_submit

2) Submit frame values:
Invoke-RestMethod -Method Post -Uri "http://localhost:3000/turn" -ContentType "application/json" -Body '{"conversationId":"c_demo","message":{"type":"ui_submit","frameId":"insurance:lookup","values":{"policyNumber":"ABC123","dob":"1988-07-01"}}}'

Expected result:
- assistant_message like "Looking up your policy..."
- no further ui_frame in this deterministic v1 path

CAN YOU RUN THIS AND SEE SOMETHING?

Yes.
- You can run the orchestrator server and see JSON responses immediately via PowerShell (commands above), Postman, or curl.
- The orchestrator currently returns structured JSON, not a standalone browser UI page.
- The visible UI rendering behavior is validated by the renderer tests and contract integration tests in airforms-renderer.
